## 简历原文（截图清单的依据，不可与之冲突）

> **AI Agent 驱动的智能刷题平台**（独立开发）| 2026.06 – 至今
> - **AI 语义搜索**：以 ChromaDB + sentence-transformers 替代传统关键词匹配，实现自然语言级语义检索（如"类似背包问题的 DP 题目"），覆盖 3,971 道算法题及 73 个标签维度
> - **三级 Agent 帮助系统**：按信息揭示量设计 Hint→Guide→Explain 递进策略——Hint 仅给方向、Guide 拆解步骤、Explain 展示思路，避免直接给答案破坏刷题学习循环；Agent 根据用户解题状态自主选择帮助层级
> - **数据工程**：解析 doocs/leetcode 多语言题解库（10+ 编程语言），结构化提取题目描述、代码模板与参考解答，建立 73 个标签到算法知识体系的全量映射，为语义搜索与 Agent 帮助提供数据基础

> ⚠️ **简历表述修正建议**：简历亮点 1 写的「ChromaDB + sentence-transformers」中，sentence-transformers 是**加载库**而非模型。实际模型为 `shibing624/text2vec-base-chinese`（中文语义模型，768 维）。建议简历改为「ChromaDB + text2vec-base-chinese 中文语义模型」或「ChromaDB + sentence-transformers（shibing624/text2vec-base-chinese）」以避免面试官追问时口径不一致。
> 依据：`backend/app/config.py` L57 `EMBEDDING_MODEL = "shibing624/text2vec-base-chinese"` + `backend/app/services/embedding.py` L6 `from sentence_transformers import SentenceTransformer`

> 对应简历亮点 2。三张图体现 Hint→Guide→Explain 递进，是简历里最核心的 Agent 能力证明。

| 文件名 | 对应功能 | 画面描述 | 要说明的技术点 |
|---|---|---|---|
| `03-agent-hint.png` | Hint 模式 · 思路方向 | 刷题界面侧栏「算法精灵」对话框：用户问"这题怎么入手"，精灵回复 1–2 句方向提示（如「想想能不能用 DP，状态代表什么？」），**右上角金币数从 20 → 19** 可见变化 | Prompt Engineering 分级提示词（`HELP_TEMPLATES["hint"]`）；金币事务一致性（LLM 成功后才扣费）；Agent 角色化对话 |

> **建议省略：真实代码评测控制台（attack 的 pass/fail 逐条结果）**

> **建议省略：多套前端风格切换（暗色 IDE / 亮色学术 / 赛博朋克）**
> 理由：这是早期风格探索阶段产物（见 MEMORY「三套静态前端风格设计」），当前已定型为杂志 + 闯关风，截图会让面试官误以为项目仍在纠结 UI 方向，削弱"AI 能力"主线。
